Building CDAT on MacOS X

  • Install the Developer Tools from the disk that came with OS X or download the latest from Apple (free online registration required).
  • Install the fink package manager. You will need root access to run fink. Be sure to read the excellent  Users Guide.
  • Edit /sw/etc/fink.conf with a text editor, add "unstable/main" to the "Trees:" line. Mine looks like this:
Trees: local/main stable/main stable/crypto  local/bootstrap  unstable/main unstable/crypto
  • Update your package descriptions from CVS by entering fink selfupdate-cvs in a terminal window. This is essential since CDAT requires the latest package versions.  When fink asks if you want to update from CVS, say yes.
  • Install cdat entering the following in a terminal window:
     fink install xfree86-rootless g77
Go get a cup of coffee (or three).  It may take several hours to build everything. You will be presented will several choices to satisfy dependencies.  You may also want to run  fink install atlas if you want access to faster LAPACK and BLAS routines in CDAT.

IMPORTANT NOTE:  You may already have XFree86 installed either from fink or from another source. Unless it is the fink xfree86-4.2 package, it will not work with CDAT.  This is because CDAT needs a threaded X11 server, and xfree86 for Darwin/MacOS X has threading turned off by default.  The fink package has it enabled.  The only other choice is to build xfree86 from source yourself and turn posix threading on.  If you already have xfree86 installed from another source (not fink), you will need to manually remove it before installing the fink package.  If you have an older version of xfree86 installed via fink, just run fink update xfree86-rootless (after updating you package descriptions from CVS).
  • Read  Running X11 on the fink web site.  You may want to install a nicer window manager (twm is the default). I recommend  icewm (fink install icewm-basic) or  Oroborus.
  • Now you can either install CDAT via the fink package, or build it manually.  To install the fink package, simply type fink install cdat in a terminal window.  You will be presented with a choice between numeric and numeric-atlas. If you have installed atlas, choose numeric-atlas, otherwise choose numeric.  The installation will take a few hours, When it's done, proceed to the last step. To build manually, follow the intervening steps.
  • Decide where you want CDAT to be installed - say for example /Users/john/cdat.
  • In a terminal window 
mkdir -p /Users/john/cdat/lib
ln -s /Users/john/cdat/lib/libreadline.a /Users/john/cdat/lib/libtermcap.a
This little trick is necessary to fool the python configure script into thinking a termcap library actually exists.  If you don't do this, the readline module will not be built.
  • setenv CPP_X11 -I/usr/X11R6/include   (this seems to be necessary for the ensuing xgks build.
  • Download CDAT and follow the instructions in the README to install in /Users/john/cdat.  
  • If you installed the atlas package, you will want to rebuild Numeric to link in the atlas libs.  Go to cdat-3.3/exsrc/build/Numeric-22.0.  Edit setup.py and delete the four lines after
sourcelist = [os.path.join('Src', 'lapack_litemodule.c')

Replace the lines
library_dirs_list = []
libraries_list = []

with
library_dirs_list = ['/sw/lib']
libraries_list = ['lapack', 'cblas', 'f77blas', 'atlas', 'g2c']
Run 
/bin/rm -rf build
/Users/john/cdat/bin/python setup.py install
  • Now you're done - download and run the tutorial scripts to make sure everything works.
